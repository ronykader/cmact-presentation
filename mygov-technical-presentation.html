SECTION 07 / 07

Shared Platform Patterns

Common architecture practices — covering authentication schemes, deployment container systems, and cross-application database security principles.

Docker & Sail CI/CD Jenkins Sanctum & OIDC Database Isolation MySQL Cluster
🛠️
Shared Patterns

Architectural Consistency

All applications are containerized with identical configurations, simplifying environment staging and streamlining deployment pipelines via Jenkins.
🐳
Container SystemDocker and Laravel Sail for unified development/staging configs
🚀
Staging PipelinesJenkins files defining checkouts, tests, and auto-build triggers
💾
MySQL IsolationEvery project utilizes its own designated database instance
🔑
Auth TokensSanctum tokens for dynamic apps and stateful JWT claim checkouts
⚙️
Deployment Standards
Containers run PHP 8.2 and Node 22. Deployments route via secure reverse proxies, with SSL termination handled at the gateway layer.
🔒
Common Practices
Strict CORS controls restrict cross-origin requests. API access relies on short-lived authentication claims.
Shared Patterns

Ecosystem Security

🛡️
OIDC SSO Federation
Citizens log in securely using the central OIDC IDP (myInfo). Federated claims are passed to individual apps without replicating credential verification databases.
🔌
Apostille Laravel 12 API Hub
All attestation processes rely on the central API Hub. Bidirectional callbacks update myGov platform status records, keeping officer dashboards segregated.
📂
eFiling — Stateless JWT + Roles
JWT stateless auth secures desk communication. spatie/laravel-permission enforces fine-grained access control across active Doptor desks and files.
🔑
myInfo IDP — Federated OpenID Connect
OpenID Provider. Laravel Passport OAuth2 Authorization Server. Issuing OIDC tokens (access, refresh, ID JWT). Rate-limiting on public registration forms.
📊
Dashboard — Doptor SSO & API Sanctum
Doptor SSO verifies government staff. Firebase JWT claims check validity. Dashboard API routes require Sanctum tokens. CSRF protection enabled.
